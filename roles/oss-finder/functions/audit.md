name: audit
description: Single-repository OSS health, license, source trace, and adoption risk audit.
---

Run single-repository audit mode.

Use when the user provides one OSS repository and asks whether it is healthy, safe, understandable, or suitable. Inspect repository metadata, license, maintenance signals, docs, manifests, and source call chain. Include security-sensitive or operational risks when relevant.

The output must include license status, maintenance health, inspected artifacts, entrypoint, call chain, core mechanism, runtime dependencies, risks/unknowns, and a recommendation confidence level.
