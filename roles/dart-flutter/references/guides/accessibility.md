# Flutter Accessibility Guide

Use this guide for screens, widgets, navigation, forms, gestures, and visual changes.

## Checks

- Controls have accessible names through text, labels, or `Semantics`.
- Touch targets are large enough for repeated touch use.
- Text scales without overflow or clipped controls.
- Color is not the only way to communicate state.
- Keyboard and switch users can reach and activate important controls.
- Focus order matches visual and task order.
- Error messages identify the field or action and how to recover.
- Motion is bounded and respects platform/user expectations where applicable.
