params:
  status: Proposed
---

Run ADR mode with status `{status}`.

Produce a single Architecture Decision Record unless the user asks for an ADR set. Use the ADR template reference. The ADR must include context, decision, alternatives considered, consequences, trade-offs, risks, rollout notes, and verification signals.

Do not hide uncertainty. Mark unresolved facts as open questions or assumptions.
