name: review
description: Architecture review mode focused on correctness, trade-offs, anti-patterns, and implementation fit.
params:
  focus: general
---

Run architecture review mode with focus `{focus}`.

Review the architecture, code boundaries, ADR, proposal, or migration plan for structural risk. Prioritize findings by severity. Each finding must name the affected decision or boundary, why it matters, the trade-off being missed, and the recommended correction.

Use the anti-pattern skill before finalizing. If there are no material findings, say so and identify residual risk.
